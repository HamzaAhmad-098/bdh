1. SSH Setup

Generate your SSH key (ED25519) and send your public key to the admin:

ssh-keygen -t ed25519 -C "your_email@example.com"
cat ~/.ssh/id_ed25519.pub

Send the single-line output via email to hamzaxdevelopers@gmail.com including:

  • Desired subdomain
  • Type of app (Node, Rails, Static HTML)
  • Any extra message or greetings
  • Note !! If User fails to verify then it automatically rejected by admin server
  • Not try to access the Root it strictly Encrypted under SSL certificate from each layer
  • If you get verified then you have your own linux account verified by Cloudflared
  • Even Admin can't Access your account Its access is just for you with a private key you generated at first step
  • You send the public key to your Admin Using "cat ~/.ssh/id_ed25519.pub" And a single-line
  • Private key is yours never share this with anyone
  • For More security if you not want not to send public key then you can encrypt This key as well using RSA Encryption Algo
  • Admin will try and send you further instructions. More you can encrypt using our encrypter with custom key
  • Available at portfolio.hamzaxdevelopers.dpdns.org
  • If any suspicious activity notice on your domain then we will send you the report and after verification youraccount will be permanently bann>

2. Install Cloudflared

Install Cloudflare tunnel on your client machine:

wget -q -O cloudflared.deb "https://github.com/cloudflare/cloudflared/releases/latest/download/cloudflared-linux-amd64.deb"
sudo dpkg -i cloudflared.deb
rm cloudflared.deb

For other like windows , MaCos you can see the official releases of cloudflare

3.Connect with server

ssh -i ~/.ssh/id_ed25519 -o ProxyCommand="cloudflared access ssh --hostname %h" username@ssh.hamzaxdevelopers.dpdns.org

Replace user name with your domain you provided to admin

5. Keep Your App Running 24/7

Once you check the App running on the domain then you can make this app forwarded to the server using systemmd

Create a systemd service so your app stays running after logout:

mkdir -p ~/.config/systemd/user
		nano ~/.config/systemd/user/myapp.service

This will be in the mail acc. to your Requirements

Example Node service:

[Unit]
Description=Username Node App
After=network.target

[Service]
WorkingDirectory=/home/username/app
ExecStart=/usr/bin/node /home/username/app/server.js
Restart=always
Environment=PORT=3001
Environment=NODE_ENV=production

[Install]
WantedBy=default.target

Then Run the File and check the status 

systemctl --user daemon-reload
systemctl --user start myapp.service
systemctl --user enable myapp.service

Check the status of app If running Congrats all done You can logout or turn off your computer Your app will be live 24/7 like netlify or vercel


systemctl --user status myapp.service
